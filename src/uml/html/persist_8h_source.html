39/* COMPATIBILITY NOTES
40 *
41 * The P_* structs (persist structs) contain all of the data for a particular
42 * data chunk. They are loaded in multiple parts, so can be rearranged without
43 * updating the db format version.
44 *
45 * The PF_* structs (persist fixed structs) contain the fixed size data for a
46 * particular data chunk. They are written to disk as is, so they must not be
47 * rearranged without updating the db format version. When adding new members,
48 * always use explicit sized datatypes ("uint32_t", not "long"), and check
49 * whether what is being added can go in an existing hole in the struct.
50 */
